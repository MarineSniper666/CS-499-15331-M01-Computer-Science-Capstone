About This Portfolio

This ePortfolio showcases my work in the CS 499 Computer Science Capstone. The primary artifact featured in this portfolio is an Inventory Management Mobile Application developed in CS 360: Mobile Architecture and Programming. The project demonstrates skills in software design and engineering, algorithms and data structures, and database integration through artifact analysis, refinement, and implementation of meaningful enhancements.

Artifact Overview

Artifact: Inventory Management Mobile Application
Course: CS 360: Mobile Architecture and Programming
Platform: Android
Database: SQLite
Primary Features: Inventory tracking, login system, SMS alerts
Focus Areas: Software Design, Algorithms, Databases
Language: Java with XML-based Android layouts
Enhancement Category: Software Design and Engineering

This artifact represents a complete mobile-based inventory management system developed using Java and Android Studio. The application allows users to create, update, and delete inventory items while maintaining persistent data through a local SQLite database. Each item includes detailed attributes such as quantity, location, manufacturing cost, retail value, and unit weight, allowing for more meaningful tracking beyond simple inventory counts. The system also includes automated SMS notification functionality that triggers when inventory levels fall below a defined threshold, enabling real-time response to low-stock conditions. This artifact was selected because it demonstrates the integration of user interface design, data management, and application logic within a functional and scalable mobile application.

Implemented Enhancements

  • Implemented structured input validation to prevent invalid or inconsistent numeric data entry
  • Added confirmation safeguards before deleting inventory items to reduce accidental data loss
  • Replaced standalone SMS controls with a centralized settings menu for improved organization
  • Integrated logout functionality into the settings menu to support proper session management
  • Introduced popup-based item editing to streamline workflow and reduce unnecessary navigation
  • Enhanced visual feedback, including a red delete button to clearly indicate destructive actions
  • Improved layout spacing and interface organization for a cleaner and more professional presentation
  • Incorporated calculation logic to extend functionality beyond basic data storage

These enhancements reflect a transition from a functional academic project to a more realistic application design. Instead of focusing only on the original implementation, the system was refined based on usability, organization, and expected user interaction.

The addition of a centralized settings menu, improved validation, popup-based editing, and clearer visual feedback demonstrates a stronger focus on both application structure and user experience. The inclusion of calculation logic also extends the system beyond basic inventory storage by making the application more practical and useful.

Together, these changes demonstrate the ability to evaluate an existing software system, identify areas for improvement, and implement meaningful refinements that better align the application with real-world expectations.

Software Design and Engineering Analysis

This artifact demonstrates the application of software design and engineering principles through both its structure and iterative refinement. The use of SQLite for persistent data storage reflects an understanding of structured data management within a mobile environment, while the separation of interface components, data handling, and logic supports maintainability and scalability.

The implemented enhancements further demonstrate the ability to evaluate and improve an existing system. One key design decision involved replacing the original color personalization feature with a centralized settings menu. This change was intentionally made to reflect real-world development scenarios where client requirements can shift unexpectedly, requiring developers to adapt existing systems to meet new priorities. The transition to a structured settings menu improves usability and aligns with standard application design practices. The addition of validation controls, improved workflows, and extended functionality through calculation logic reflects a continued focus on both reliability and practical usability.

Overall, this project demonstrates the ability to design, develop, and refine a complete software system while balancing technical implementation with user-centered design considerations. It highlights not only the creation of a working application, but also the ability to improve that application in response to real-world expectations.

Algorithms and Data Structures

This section focuses on how algorithmic logic and structured data handling were applied within the inventory management application. While the system does not rely on complex computational algorithms, it demonstrates practical algorithmic thinking through efficient data processing, validation, and real-time decision making.

One key area of improvement involved input validation and data consistency. Structured validation logic was implemented to ensure that all numeric values, such as quantity, cost, and weight, are properly formatted and within expected ranges before being committed to the database. This reduces the risk of corrupted or inconsistent data and ensures that all operations performed on the dataset remain reliable.

The application also incorporates conditional logic to support real-time system behavior. For example, inventory levels are evaluated during create and update operations, triggering SMS notifications when quantities fall below a defined threshold. This demonstrates the use of decision-based algorithms that respond dynamically to changing data conditions.

Additionally, the use of structured data within the application reflects an understanding of data organization and access. Inventory items are managed in a consistent format, allowing for predictable retrieval, updates, and display within the user interface. These design decisions improve overall efficiency and maintainability while ensuring that the application remains responsive and user-friendly.

Database Design and Integration

The database serves as the foundation for persistent data storage, allowing users to maintain and manage inventory records across sessions.

The database schema was designed to store structured inventory data, including item name, quantity, location, case size, manufacturing cost, retail cost, and unit weight. This structured approach ensures consistency across all records and supports reliable data retrieval and updates.

Enhancements were made to improve data integrity and system reliability. Input validation was integrated with database operations to prevent invalid data from being stored, reducing the likelihood of errors during retrieval or processing. Additionally, delete operations were safeguarded with confirmation prompts, ensuring that data removal is intentional and controlled.

The interaction between the application and the database demonstrates an understanding of CRUD operations, including creating new records, reading existing data, updating entries, and safely deleting items. These operations are handled efficiently to maintain performance while ensuring that the user interface remains responsive.

Overall, the database design supports a stable and scalable system by ensuring data consistency, reliability, and efficient access. These improvements reflect the importance of well-structured data management in real-world applications.
